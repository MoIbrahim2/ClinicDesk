# ClinicDesk Hackathon Project

This repository now follows a BMAD working structure:

- `B`usiness: scope, requirements, priorities, and sprint focus
- `M`odel: use cases, domain rules, entities, and database design
- `A`pplication: APIs, backend modules, frontend pages, and integration work
- `D`elivery: wireframes, roadmap, task slicing, testing, and demo readiness

## Recommended Working Order

1. Lock MVP from `Business`.
2. Define entities and rules from `Model`.
3. Build backend and frontend slices in `Application`.
4. Validate flows, demo scope, and sprint status in `Delivery`.

## Suggested First Build Slice

1. Authentication and roles
2. Patient registration and search
3. Appointment scheduling with conflict detection
4. Visit/examination recording
5. Prescription creation
6. Invoice and payment recording
7. Role dashboards
